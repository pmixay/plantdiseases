### Архитектура

```
Android (Kotlin)                    Python Server (FastAPI)
┌────────────────────┐              ┌──────────────────────────┐
│ CameraX → фото     │    HTTP      │ POST /api/analyze        │
│ Retrofit → сервер   │ ──────────► │   → валидация (тип, вес) │
│ Room → локальная БД │              │   → CNN inference        │
│ Material 3 UI       │    JSON      │   → DISEASES_DATABASE    │
│ 4 вкладки:          │ ◄────────── │   → двуязычный ответ     │
│  Сканер/Галерея/    │              │                          │
│  Справка/Профиль    │              │ GET /api/health          │
└────────────────────┘              └──────────────────────────┘
```

---

## Блок 1 — Обучение модели (P0)

Модель готова к обучению. Скрипт настроен, нужен датасет.

- **P0** — Скачать PlantVillage dataset и разбить на 15 целевых классов
- **P0** — Обучить модель: `python train.py` (≥ 75% accuracy)
- **P0** — Проверить confusion matrix на валидации, убедиться в отсутствии критичных ошибок
- **P1** — Дополнить датасет фотографиями комнатных растений (iNaturalist, собственные)
- **P2** — Сравнить с EfficientNet-B0

---

## Блок 2 — Тестирование и финализация (P0)

- **P0** — Функциональное тестирование на физическом устройстве и эмуляторе
- **P0** — Тестирование полного цикла: камера → анализ → результат → галерея
- **P0** — Проверка двуязычности (переключение RU/EN)
- **P1** — Тестирование на реальных фото комнатных растений (не из датасета)
- **P1** — Замер метрик: accuracy, latency ответа сервера, crash rate
- **P2** — Подготовка signed APK для демонстрации
- **P2** — Опрос тестовой группы (≥ 70% удобство)

---

## Блок 3 — Надёжность (P1)

- **P1** — Обработка потери сети при анализе (retry с индикацией)
- **P2** — Offline-режим: TFLite на устройстве (fallback если сервер недоступен)
- **P2** — Фильтр «не растение»: предупреждение если фото не содержит листья
- **P3** — Splash screen (Android 12+ SplashScreen API)
- **P3** — Фильтрация в галерее (больные / здоровые)
- **P3** — Поиск по справочнику

---

## Блок 4 — Дополнительные улучшения (P2-P3)

- **P2** — Тёмная тема (Material 3)
- **P2** — Кэширование на сервере (Redis)
- **P2** — Логирование запросов в файл на сервере
